## 事業リスク

### ① AI対話品質リスク（発生確率: 高 / 影響度: 高）

**リスク内容:**
AI対話でのミッション生成が表面的・画一的になり、転職者が「これ本当に意味ある？」と感じる。パーソナライズの質が低いとサービスの根幹が揺らぐ。

**想定シナリオ:**
- 同じ企業の転職者全員に似たようなミッションが生成される
- 「あなたの強みはコミュニケーション力です」のような汎用的な回答ばかり
- 業界知識の不足で的外れな提案をする

**対策:**
| 対策 | 実施時期 | コスト | 効果 |
|------|---------|--------|------|
| プロンプトエンジニアリング専任配置 | MVP〜 | ¥500K/月 | 対話品質の底上げ |
| 人事プロのレビュー体制 | MVP〜 | ¥200K/月 | 専門性の担保 |
| 転職者フィードバックループ | ローンチ後 | 開発工数 | 継続的改善 |
| ファインチューニング用データ蓄積 | Year 1〜 | ¥1M | パーソナライズ精度向上 |
| 人間メンターのフォールバック | ローンチ後 | ¥300K/月 | AI限界時の補完 |

**残存リスク:** 中。継続的な改善で品質は上がるが、AI特有の限界は存在し続ける。

### ② データ漏洩リスク（発生確率: 高 / 影響度: 高）

**リスク内容:**
転職者の個人情報（現職情報、価値観、強み・弱み）と企業の内部情報（課題、組織構造）の両方を扱うため、漏洩時の影響が甚大。

**想定シナリオ:**
- AI API経由でのデータ流出（学習データに混入）
- 不正アクセスによる転職者データの流出
- 企業の機密情報（組織課題等）が競合他社に渡る
- 転職者の「現職への不満」が現職企業に伝わる

**対策:**
| 対策 | 詳細 |
|------|------|
| AI APIのデータ保持設定 | Anthropic/OpenAI共に学習に使用しない契約を締結 |
| データ暗号化 | 保存時: AES-256、通信時: TLS 1.3 |
| RLS（Row Level Security） | Supabaseで企業間・ユーザー間のデータ分離 |
| SOC2 Type II取得 | Year 2目標で信頼性を客観的に証明 |
| ペネトレーションテスト | 半年に1回、外部業者に依頼 |
| データ保持期限 | 入社後6ヶ月で対話データ自動削除 |

### ③ 競合参入リスク（発生確率: 中 / 影響度: 高）

**リスク内容:**
大手HRテック企業（SmartHR, HRMOS, カオナビ等）が類似機能を自社プロダクトに追加する可能性。

**想定競合:**

| 競合 | 脅威レベル | 参入可能性 | 差別化 |
|------|-----------|-----------|--------|
| SmartHR | 高 | 中 | 彼らは労務管理がコア。オンボーディングは周辺機能 |
| HRMOS | 高 | 高 | 採用管理の延長で入りやすい。ただしAI対話は別領域 |
| カオナビ | 中 | 中 | タレマネの文脈で可能性あるが優先度低い |
| 海外（Enboarder等） | 中 | 低 | 日本市場のローカライズコストが障壁 |
| スタートアップ | 低〜中 | 高 | 同じ市場を狙う新規参入者 |

**対策:**
- **先行者優位の確立**: Year 1で大手企業20社の導入実績を作る
- **データモート**: 蓄積された対話データ・ミッションパターンが参入障壁に
- **企業文化コンテンツの独自性**: 顧客企業と共同制作したコンテンツは真似できない
- **NPS経由の口コミ**: 人事コミュニティでの評判を早期に確立

### ④ 企業導入障壁リスク（発生確率: 高 / 影響度: 中）

**リスク内容:**
「入社前にそこまでやる必要ある？」という企業の認識。人事部門の工数・予算の優先順位。

**想定される反対意見:**
- 「うちのオンボーディングで十分」
- 「入社前に企業情報を出しすぎるのはリスク」
- 「転職者のエンゲージメントは入社後に上がるもの」
- 「予算が取れない」

**対策:**
| 対策 | 詳細 |
|------|------|
| ROI算出ツール | 離職率改善 × 採用コスト削減で具体的な金額を提示 |
| 無料トライアル（3名まで） | まず使ってもらう。効果を体感させる |
| ケーススタディ蓄積 | 導入企業の定着率改善データを公開 |
| 人事向けセミナー | 「入社前エンゲージメント」の重要性を啓蒙 |
| 段階的導入 | まず1部署で試して、全社展開を提案 |

### ⑤ 法規制リスク（発生確率: 高 / 影響度: 中）

**リスク内容:**
個人情報保護法、AI規制の強化。特にAIによる人材評価に関する法規制が世界的に強まっている。

**関連法規:**
| 法規 | リスク | 対策 |
|------|--------|------|
| 個人情報保護法 | 対話データの取扱い | プライバシーポリシー整備、同意取得フロー |
| EU AI Act | AIによる雇用決定への規制 | 「評価」ではなく「支援」の位置付けを明確化 |
| 職業安定法 | 人材紹介に該当する可能性 | 法務チェック、必要に応じ許認可取得 |

### ⑥ 景気悪化リスク（発生確率: 中 / 影響度: 中）

**リスク内容:**
景気後退で中途採用市場が縮小。企業が採用を凍結すると新規ユーザー（転職者）が減る。

**対策:**
- 不景気時は「厳選採用 → 1人あたりの定着が重要」の訴求に切り替え
- B2Cフリーミアムで個人ユーザーを獲得（転職準備ツールとして）
- コスト構造を変動費中心に（AI APIは従量課金）

### ⑦ 技術負債リスク（発生確率: 高 / 影響度: 低）

**リスク内容:**
MVP開発の速度重視で技術負債が蓄積。スケール時にリファクタリングが必要に。

**対策:**
- MVP段階からTypeScript strictモード
- テストカバレッジ70%以上の目標
- Phase 2開始前にアーキテクチャレビュー実施

### ⑧ チーム離脱リスク（発生確率: 中 / 影響度: 低）

**リスク内容:**
初期メンバーの離脱でプロダクト開発が遅延。

**対策:**
- ドキュメント文化の徹底（Notion）
- コードレビュー必須化
- 株式報酬（SO）による長期インセンティブ
