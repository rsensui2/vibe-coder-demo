## 事業リスク

### 1. 金融商品取引法（金商法）規制リスク 🔴

**概要:** 株式の売買推奨は金商法上の「投資助言・代理業」に該当する可能性がある。無登録での投資助言は法的リスクが極めて高い。

**詳細分析:**
- 「この株を買うべき」と断定的に推奨する行為は、投資助言業登録が必要
- 登録には資本金500万円以上、コンプライアンス体制整備、金融庁への届出が必要
- 違反した場合: 5年以下の懲役 or 500万円以下の罰金

**対策:**
| 対策 | 実施時期 | コスト |
|------|---------|--------|
| 法務顧問（金融法専門）との契約 | MVP前 | ¥50万/年 |
| 「情報提供」と「助言」の境界を明確化 | MVP前 | - |
| 免責事項・利用規約の整備 | MVP前 | ¥30万 |
| 投資助言業登録の検討 | Year 2 | ¥200万+ |
| 表現のガイドライン策定 | MVP前 | - |

**具体的な対応方針:**
- MVP段階: 「AIによる分析結果の提供」に留め、「買い」「売り」の断定的推奨を避ける
- トレンドスコアは「参考情報」として提供、投資判断は自己責任である旨を明示
- Year 2以降: ユーザー数が一定規模に達した段階で投資助言業登録を検討

### 2. データ取得制限リスク 🟠

**概要:** 株価・財務データのAPI利用には制限やコストがかかる。特に東証のリアルタイムデータは高額。

**詳細:**
- 東証リアルタイム株価: 情報ベンダー契約が必要（月額数十万円〜）
- 15分ディレイデータ: Yahoo!ファイナンスAPI等で取得可能だが、利用規約に制約あり
- 決算データ: EDINETは無料だがパース処理が必要
- EC固有KPI（GMV等）: 各社IRから手動収集が必要な場合あり

**対策:**
- Phase 1: 15分ディレイデータ + EDINET無料データで開始
- Phase 2: 有料データベンダー（Refinitiv, Bloomberg等）との契約検討
- EC固有KPIは決算短信PDFからAI抽出する仕組みを構築
- データソースの冗長化（複数API併用）

### 3. ユーザー獲得困難リスク 🟠

**概要:** ニッチ市場（EC業界×株式投資）のため、ターゲット母数が限定的。

**CAC試算と課題:**

| チャネル | 想定CAC | 月間獲得見込み | 課題 |
|---------|---------|--------------|------|
| SEO/コンテンツ | ¥1,000 | 100名 | 効果発現まで6ヶ月 |
| X/SNS運用 | ¥2,000 | 50名 | 属人的、スケールしにくい |
| Google広告 | ¥5,000 | 200名 | 競合が少なくCPC安い可能性 |
| 投資系メディア提携 | ¥3,000 | 150名 | 記事広告、アフィリエイト |
| IR担当者口コミ | ¥0 | 10名 | 時間がかかる |

**対策:**
- EC業界メディア（ECのミカタ、ネットショップ担当者フォーラム等）との提携
- 投資系インフルエンサーとのコラボ
- 無料版の充実でオーガニック流入を最大化
- 決算シーズンに合わせたPR戦略

### 4. 競合参入リスク 🟡

**想定される競合の動き:**

| 競合候補 | 参入可能性 | 脅威レベル |
|---------|-----------|-----------|
| マネックス証券 | 中 | 高（既存顧客基盤） |
| バフェット・コード | 高 | 中（セクター特化は未対応） |
| moomoo証券 | 低 | 中（米国株メイン） |
| 大手メディア（東洋経済等） | 低 | 低（SaaS型への転換は困難） |
| 新規スタートアップ | 中 | 中（同じ着眼点の可能性） |

**対策:**
- EC業界のドメイン知識を最大の参入障壁とする
- 独自データ（EC固有KPI、トレンドスコア）の蓄積で差別化
- ユーザーコミュニティの構築で switching cost を高める
- 特許出願（スコアリングアルゴリズム）

## 技術リスク

### 5. AI精度・ハルシネーションリスク 🟡

**概要:** AIによる投資提案が不正確な場合、ユーザーの損失につながり、信頼を失う。

**対策:**
- AIの出力には必ず根拠データ（ソース）を表示
- バックテスト結果を開示し、過去のパフォーマンスを検証可能にする
- 「AIスコアの的中率」を定期的に計測・公開
- 人間のアナリストによるレビューレイヤーを設置
- 免責事項の明確な表示

### 6. セキュリティリスク 🟠

**概要:** ユーザーの投資情報・ポートフォリオデータは高度なプライバシー情報。

**対策:**
| 対策 | 実施 |
|------|------|
| データ暗号化（at rest / in transit） | AES-256 / TLS 1.3 |
| 認証 | OAuth 2.0 + MFA |
| アクセス制御 | RBAC + 最小権限原則 |
| 監査ログ | 全操作の記録・90日保持 |
| 脆弱性診断 | 四半期ごとの外部診断 |
| SOC 2 Type II | Year 2で取得目標 |

## 法的リスクまとめ

| 法令 | 関連事項 | 対応状況 |
|------|---------|---------|
| 金融商品取引法 | 投資助言業の登録要否 | 法務顧問に確認中 |
| 個人情報保護法 | ユーザーデータの取り扱い | プライバシーポリシー策定 |
| 景品表示法 | パフォーマンス表示の適正性 | 根拠データの保持 |
| 不正競争防止法 | 競合データの利用制限 | 公開情報のみ使用 |
| 著作権法 | ニュース記事の引用 | 引用ルールの遵守 |
