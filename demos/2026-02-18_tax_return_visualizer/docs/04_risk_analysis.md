## 詳細リスク分析と対策

### 🔴 最優先リスク

#### 1. 税法改正への追従遅れ

**リスク内容:**
日本の税法は毎年12月の税制改正大綱で変更が発表され、翌年1月から適用されることが多い。確定申告シーズン（2-3月）までに新税法を正確に実装する必要がある。

**影響:**
- 誤った税額を表示 → ユーザーの申告ミスに直結
- 信頼の失墜 → 致命的なレピュテーションダメージ
- 法的リスク → 損害賠償請求の可能性

**対策:**
| 対策 | 実施時期 | コスト |
|------|---------|--------|
| 税理士顧問契約（2名以上） | 常時 | 月額30万円 |
| 税法変更の自動モニタリングシステム | Q2 | 開発2ヶ月 |
| 税制改正大綱リリース→実装完了を30日以内にする体制 | 毎年12月 | 人件費増 |
| ユーザーへの改正影響の自動通知 | Q3 | 開発1ヶ月 |

#### 2. 計算ロジックの誤り

**リスク内容:**
所得税・住民税の計算は、控除の順序・端数処理・特例適用など複雑な条件分岐が多い。リアルタイム計算の速度を優先するあまり、精度が犠牲になる可能性。

**影響:**
- ユーザーが誤った金額で申告
- 税務署からの指摘 → ユーザーからのクレーム
- 集団訴訟リスク

**対策:**
| 対策 | 詳細 |
|------|------|
| 国税庁の計算例をテストケースに | 公開されている全計算例をE2Eテスト化 |
| 税理士によるダブルチェック体制 | 主要パターン100ケースを毎年検証 |
| 免責事項の明確化 | 「本サービスは参考値であり、最終的な税額は税務署にご確認ください」 |
| 確定申告書との突合機能 | e-Tax提出前に国税庁の計算結果と比較 |
| バグバウンティプログラム | 計算ミスの報告に報奨金 |

### 🟡 高優先度リスク

#### 3. 個人情報漏洩

**リスク内容:**
確定申告データは収入・資産・家族構成など極めてセンシティブな個人情報の塊。

**対策:**
- SOC 2 Type II認証の取得（2年目）
- データの暗号化（転送中: TLS 1.3、保存時: AES-256）
- 個人情報は日本国内のデータセンターに限定
- 年次ペネトレーションテスト実施
- 従業員のセキュリティ研修（四半期）
- マイナンバーは保持しない設計（入力→即e-Tax送信→破棄）

#### 4. 競合の類似機能追加

**リスク内容:**
freee、マネーフォワード、弥生がリアルタイム可視化機能を追加する可能性。

**対策:**
- **スピード:** 12ヶ月のリードタイムを活かし、UXの磨き込みで差別化
- **特化:** 確定申告に100%特化することで、汎用会計ソフトにはない深さを実現
- **ネットワーク効果:** 税理士マッチングプラットフォームの両面市場を早期構築
- **データモート:** 匿名化された申告データの蓄積による節税提案の精度向上

#### 5. 確定申告シーズンの集中負荷

**リスク内容:**
確定申告期間（2/16〜3/15）にアクセスが年間の80%以上集中。

**対策:**
- オートスケーリング設計（Vercel Edge + Supabase）
- CDN活用で静的コンテンツを分散
- 負荷テスト: ピーク想定の3倍で実施
- 段階的ロールアウト（地域別・ユーザー層別）

#### 6. 税理士法との抵触

**リスク内容:**
税理士法第52条: 税理士でない者が税務相談を行うことは禁止。AIアドバイザー機能が「税務相談」に該当する可能性。

**対策:**
- 法務チームによる機能リリース前レビュー
- AIアドバイスは「一般的な情報提供」の範囲に限定
- 個別具体的なアドバイスは税理士マッチングに誘導
- 税理士会との事前協議・連携

## コンティンジェンシープラン

| シナリオ | 発動条件 | アクション |
|---------|---------|-----------|
| 計算バグ発覚 | 影響ユーザー100人以上 | 即時通知→修正値提示→申告修正サポート |
| データ漏洩 | 個人情報流出の疑い | サービス一時停止→調査→当局報告（72時間以内） |
| e-Tax障害 | 国税庁側の障害 | PDF出力への誘導→復旧後の再提出サポート |
| 資金ショート | ランウェイ6ヶ月以下 | Freeプラン制限→有料プラン値上げ→緊急増資 |

## 法的・コンプライアンス対応

| 法規制 | 対応 |
|--------|------|
| 個人情報保護法 | プライバシーポリシー整備、Pマーク取得 |
| 税理士法 | AI機能の範囲制限、税理士連携 |
| 電子帳簿保存法 | スキャナ保存要件への対応 |
| JISQ15001 | 個人情報マネジメントシステム構築 |
