# InfoRadar — Business Model

## 収益構造

### SaaS サブスクリプション（主軸）

| プラン | 月額 | 年額（2ヶ月無料） | 対象 |
|---|---|---|---|
| **Free** | ¥0 | ¥0 | 個人・お試し |
| **Pro** | ¥1,480 | ¥14,800 | 個人ヘビーユーザー |
| **Team** | ¥4,980/人 | ¥49,800/人 | スタートアップ・中小企業 |
| **Enterprise** | 要相談 | 要相談 | 大企業・金融機関 |

**Unit Economics（Pro プラン）：**
- 月次チャーンレート目標：**3.5%**
- LTV：¥1,480 ÷ 3.5% = **¥42,286**
- CAC目標：**¥8,000**（LTV/CAC = 5.3倍）

---

### API収益（第二の柱）

情報収集APIをB2B向けに公開。

```
API料金体系：
- 基本API: ¥0.5 / リクエスト
- 要約API: ¥2 / リクエスト
- バルクプラン: 月100万req → ¥300,000
```

想定顧客：調査会社・メディア・BI ツールベンダー

---

### データインサイト（第三の柱・年1以降）

匿名化された「何が注目されているか」のトレンドデータを企業に販売。

- トピック別注目度スコア
- 業界別情報需要レポート（月次）
- 競合モニタリングレポート

## コスト構造

### 変動費（MRR比率）

| カテゴリ | 比率 | 内訳 |
|---|---|---|
| AI API費 | 18% | GPT-4o + Claude 要約コスト |
| インフラ | 8% | AWS ECS + RDS + CloudFront |
| クローリング | 4% | Playwright クラスタ |
| 通知配信 | 3% | Resend / LINE / Slack API |

**粗利率目標：67%**

### 固定費（月次）

| カテゴリ | 金額 |
|---|---|
| 人件費（5名） | ¥3,500,000 |
| オフィス | ¥300,000 |
| SaaS ツール | ¥200,000 |
| マーケ（SEO/広告） | ¥500,000 |
| **合計** | **¥4,500,000** |

---

## 損益分岐点分析

**月次固定費：¥4,500,000**
**平均Revenue per User（Pro換算）：¥1,480 × 粗利率67% = ¥991**

**BEP = ¥4,500,000 ÷ ¥991 = 約 4,541ユーザー**

Year 1 Q3（β開放後6ヶ月）での達成を目標とする。

---

## グロース戦略

### フリーミアム × バイラル

1. **Free → Pro 転換の設計**
   - Freeは週1回配信 → Pro は毎日配信（使い慣れるとFreqが足りなくなる設計）
   - 「あなたが見落とした重要ニュース（先週3件）」バナーで有料化誘導

2. **バイラルメカニズム**
   - InfoSummaryのシェア機能（Twitter/X, LinkedIn）
   - 「このブリーフィングをInfoRadarで作成」クレジット表示
   - Team招待でPro 1ヶ月無料プレゼント

3. **コミュニティ × SEO**
   - 業界別「What's Trending」週次レポートを無料公開（SEO流入）
   - SlackコミュニティとのConnectでユーザー同士の情報交換
