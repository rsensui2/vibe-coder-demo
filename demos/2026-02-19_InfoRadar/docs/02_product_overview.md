# InfoRadar — Product Overview

## プロダクトビジョン

> 「知るべき情報を、あなたのタイミングで、一次情報から。」

情報過多の時代に、本当に重要な「一次情報」だけを届ける。それがInfoRadarの存在意義です。

---

## コア機能

### 1. トピック設定（My Radar）

ユーザーは自分の「レーダー」を設定します。

```
トピック例：
- "AI Overview (AIO)" ← 一次情報源: Google, OpenAI, Anthropic公式ブログ
- "BCI / 脳波技術" ← 一次情報源: Nature, IEEE, 各社プレスリリース
- "EV市場 日本" ← 一次情報源: 国交省, 各自動車メーカーIR
- "スタートアップ資金調達" ← 一次情報源: 金融庁EDINET, TechCrunch
```

**設定項目：**
- トピック名（自由入力）
- 一次情報源の優先度（公式 > 論文 > メディア）
- 言語フィルター（日本語 / 英語 / 両方）
- 通知頻度（毎日 / 週3回 / 週1回）

---

### 2. AIクローリング＆フィルタリング

**一次情報の定義と自動判別：**

| ソースタイプ | 優先度 | 例 |
|---|---|---|
| 公式ブログ・プレスリリース | ★★★★★ | Google AI Blog, OpenAI Blog |
| 査読論文・学術誌 | ★★★★★ | arXiv, Nature, IEEE |
| 公的機関・政府 | ★★★★☆ | 総務省, 内閣府, SEC |
| 企業IR・決算資料 | ★★★★☆ | EDINET, SEC EDGAR |
| 主要メディア（一次取材） | ★★★☆☆ | Nikkei, Reuters |
| まとめサイト・ブログ | ★☆☆☆☆（除外） | Qiita要約, まとめブログ |

---

### 3. AI要約エンジン（InfoSummary）

単なる要約ではなく、**「なぜ今、あなたに重要か」**を説明します。

```
【InfoSummary サンプル】
📡 トピック: AI Overview (AIO)

🔥 今週最重要 ─────────────────
Google、検索結果にAIOを全言語展開
📄 出典: Google Search Central Blog（2025/02/17）

▶ 何が変わった？
検索結果上部に表示されるAI生成の要約（AIO）が、
英語圏以外の全言語で正式展開。日本語も対象。

▶ なぜあなたに重要？
SEOの常識が変わります。従来のクリック流入が
平均38%減少（Google社内データ）。対応必須。

▶ 次のアクション
- AIOで自社コンテンツが引用されるか確認
- 構造化データのマークアップ見直しを推奨

🔗 原文: https://developers.google.com/...
```

---

### 4. 通知チャネル

- **メール**（HTML / テキスト選択可）
- **Slack連携**（チャンネル指定 / DM選択）
- **LINE通知**
- **iOS / Androidアプリ**（プッシュ通知）
- **Webhook**（Notion / Zapier等への連携）

---

### 5. 学習・パーソナライズ

- 記事を読む → スキップ → 保存の行動を学習
- **3週間でパーソナライズスコア85%到達**
- 「あなたが見落とした重要ニュース」のリマインド機能
- 同じトピックを追うメンバーとのソーシャル機能（Team プランのみ）

## 技術スタック

```yaml
フロントエンド: Next.js 15 + TypeScript
バックエンド: Python (FastAPI) + Node.js
AI/LLM: GPT-4o + Claude 3.5 Sonnet（要約エンジン）
クローリング: Playwright + Crawlee（JS実行対応）
一次情報判定: カスタムMLモデル（精度93%）
DB: PostgreSQL + Redis（キャッシュ）
通知: Resend（メール）/ Slack API / LINE Messaging API
インフラ: AWS ECS + RDS + CloudFront
```
