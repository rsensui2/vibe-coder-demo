# リスク分析：50代からのAI講座

## リスクマトリクス

```
影響度
  高 │  ④技術変化   ①AI規制強化
     │              ②競合参入
  中 │  ⑥講師不足   ③解約率高騰
     │  ⑧個人情報   ⑤API障害
  低 │  ⑨為替影響   ⑦ブランド毀損
```

## 事業リスク

### ① AI規制強化リスク（影響: 高 / 発生確率: 高）

**概要:** 日本政府および国際社会でのAI規制が強化され、教育コンテンツの内容変更を余儀なくされるリスク。

**具体的シナリオ:**
- EU AI Act の日本版施行により、AI利用に関する教育義務化 → むしろ追い風にもなりうる
- 特定のAIサービス（ChatGPT等）の日本国内利用制限
- 医療・金融等の分野でAI活用の規制強化 → カリキュラム改訂必要

**対策:**
- カリキュラムを特定AIサービスに依存しない設計にする
- 複数AIプロバイダー（OpenAI、Anthropic、Google）に対応
- 規制動向を月次でモニタリング、法務顧問と連携
- 「AIリテラシー」という普遍的スキルにフォーカス（ツールは変わっても考え方は同じ）

### ② 競合参入リスク（影響: 高 / 発生確率: 高）

**概要:** 大手教育企業やIT企業がシニア向けAI教育市場に参入するリスク。

**想定競合:**
| 競合カテゴリ | 具体例 | 脅威度 |
|------------|--------|--------|
| 大手通信教育 | ユーキャン、ベネッセ | ★★★★★ |
| IT教育 | Schoo、Udemy Japan | ★★★★☆ |
| シニア向けサービス | ハルメク、趣味人倶楽部 | ★★★★☆ |
| 自治体直営 | 各市区町村のデジタル教室 | ★★★☆☆ |
| AI企業直営 | OpenAI公式、Google公式 | ★★★☆☆ |

**対策:**
- 先行者優位を確立（シニア×AI特化のブランドポジション）
- コミュニティの粘着性で差別化（単なる講座ではなく仲間づくり）
- 講師ネットワークの構築（認定講師制度）
- B2B法人チャネルの早期開拓（スイッチングコスト構築）

### ③ 解約率高騰リスク（影響: 中 / 発生確率: 高）

**概要:** シニア層の継続学習率が想定を下回り、月次解約率が高止まりするリスク。

**原因分析:**
- 学習習慣が定着しない
- 成功体験が不足
- 周囲に相談相手がいない
- 健康上の理由で受講が困難になる

**対策:**
- オンボーディング強化（初月は週3回のフォローアップ）
- AIバディ制度（受講生同士のペアリング、互いの進捗確認）
- ゲーミフィケーション（バッジ、連続学習日数、ランキング）
- 「休会制度」導入（退会ではなく一時停止を促す）
- 月次アンケートで離脱予兆を検知、個別フォロー

### ④ 技術変化リスク（影響: 高 / 発生確率: 低）

**概要:** AI技術の急激な変化により、カリキュラムが陳腐化するリスク。

**対策:**
- カリキュラムを「考え方」と「操作方法」に分離
- 操作方法部分はモジュール化し、四半期ごとに更新
- 「AI最新情報」ニュースレターを週1配信
- 講師向け月次研修で最新技術をキャッチアップ

## 技術リスク

### ⑤ AI API障害・値上げリスク（影響: 中 / 発生確率: 高）

**概要:** 利用するAI APIの障害や大幅値上げにより、サービス提供に支障をきたすリスク。

**過去事例:**
- 2024年: OpenAI APIの大規模障害（3時間）
- 2025年: Claude API料金改定
- GPT-4 API料金は2023年の発売時から段階的に低下傾向

**対策:**
- マルチプロバイダー構成（OpenAI / Anthropic / Google の3社並行）
- APIフォールバック機能の実装（1社障害時に自動切替）
- API利用コストの上限アラート設定
- ローカルLLM（Ollama + Llama）でのバックアップ検証

### ⑥ 講師不足リスク（影響: 中 / 発生確率: 低）

**概要:** シニア層に教えられるAI講師の確保が困難になるリスク。

**対策:**
- 講師養成プログラムの内製化
- 卒業生からの講師候補育成パイプライン
- 動画コンテンツによる非同期学習の比率を上げる
- AIアシスタントによる質問自動回答（講師の負荷軽減）

### ⑧ 個人情報漏洩リスク（影響: 中 / 発生確率: 低）

**概要:** 受講生の個人情報やAI利用履歴が漏洩するリスク。

**対策:**
- Supabase RLS（Row Level Security）による厳格なアクセス制御
- 受講生のAIチャット履歴は端末ローカルに保存（サーバーに保存しない）
- 個人情報取扱規程の策定、Pマーク取得検討
- セキュリティ監査の年次実施
- インシデント対応マニュアルの整備

## 法的リスク

### ⑦ ブランド毀損リスク（影響: 低 / 発生確率: 中）

**概要:** 受講生がAIを誤用し、トラブルが発生することでブランドが毀損するリスク。

**想定シナリオ:**
- 受講生がAIの医療アドバイスを鵜呑みにして健康被害
- AIの誤情報を拡散してしまう
- AIで作成した文書の著作権問題

**対策:**
- カリキュラムに「AIの限界と注意点」を必須セクションとして組み込む
- 「AIの回答は必ず自分で確認しましょう」の繰り返し教育
- 免責事項の明確な掲示
- 利用規約でAI誤用に関する責任範囲を明確化

### ⑨ 為替・海外サービス価格リスク（影響: 低 / 発生確率: 低）

**概要:** AI APIの料金がUSD建てのため、円安進行時にコスト増加するリスク。

**対策:**
- API利用コストを売上の5%以内に抑える設計
- 年次で価格改定を検討
- 国産LLMの活用検討（Preferred Networks等）
