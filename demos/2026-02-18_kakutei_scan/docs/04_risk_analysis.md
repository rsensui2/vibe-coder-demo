## 詳細分析

### 🔴 リスク1: OCR精度不足

**シナリオ**: 手書き領収書、かすれた印字、特殊フォーマットでOCR精度が低下し、ユーザーの修正負担が増加。

**影響**:
- ユーザー離脱率の上昇
- 「手入力の方が早い」という評価
- 信頼性への致命的なダメージ

**対策**:
1. **マルチエンジン**: Google Cloud Vision + Azure AI Document Intelligence を併用し、高い方を採用
2. **後処理AI**: LLMで文脈補完（「コンビニ」+「¥108」→ 消耗品費と推定）
3. **ユーザーフィードバックループ**: 修正データを蓄積し、精度を継続改善
4. **段階的自動化**: 確信度95%未満はユーザー確認、99%以上は完全自動
5. **精度保証**: 「読み取れなかった場合は手入力サポート」を明示

### 🔴 リスク2: 法改正への追従遅れ

**シナリオ**: 税制改正、インボイス制度の変更、電子帳簿保存法の要件変更に対応が遅れる。

**影響**:
- 生成した申告書が不正確
- ユーザーのペナルティリスク
- 法的責任問題

**対策**:
1. **税務アドバイザリーボード**: 現役税理士2-3名を顧問に招聘
2. **法改正モニタリング**: 国税庁RSSフィード+パブリックコメント監視の自動化
3. **テンプレート分離**: 申告書ロジックをテンプレート化し、改正時はテンプレート差し替えで対応
4. **バージョニング**: 年度別のルールセットを管理
5. **免責事項**: 「最終確認は税理士に相談」の明示

### 🔴 リスク3: 大手参入

**シナリオ**: freee、マネーフォワード、弥生がOCR+AI仕分け機能を強化し、カクスキャンの差別化が困難に。

**影響**:
- 価格競争の激化
- ユーザー獲得コストの上昇
- 市場シェア獲得の困難

**対策**:
1. **スピード**: 大手より速い開発サイクル（Vibe Coding活用）
2. **UX特化**: 「シンプルさ」で勝負。大手は機能過多になりがち
3. **ニッチ攻略**: 特定業種（飲食店、フリーランスエンジニア等）に深く特化
4. **税理士チャネル**: 税理士事務所との密な連携で、切り替えコストを高める
5. **M&A出口**: 差別化困難な場合は大手への売却をExit戦略に含める

### 🟡 リスク4: セキュリティインシデント

**シナリオ**: 領収書画像や経費データの漏洩。金融データを扱うため、インシデント時のダメージが大きい。

**対策**:
1. **暗号化**: 保存時AES-256、通信時TLS 1.3
2. **アクセス制御**: RBAC + 最小権限原則
3. **監査ログ**: 全操作を記録、90日保持
4. **ペネトレーションテスト**: 四半期ごとに外部実施
5. **インシデント対応計画**: 72時間以内の通知体制
6. **保険**: サイバーセキュリティ保険加入

### 🟡 リスク5: ユーザー獲得コスト高騰

**対策**:
1. **コンテンツマーケティング**: 確定申告Tips、節税ノウハウのSEO記事
2. **確定申告シーズン集中**: 1-3月にリスティング広告集中投下
3. **税理士紹介プログラム**: 顧問先への推奨でCACゼロ獲得
4. **口コミ促進**: 「友達紹介で1ヶ月無料」キャンペーン
5. **フリーミアム**: 無料プランでの体験 → 申告書生成で有料転換
