AIが生活の一部になった男のリアルな一日


はじめに：「働きたくねえ」と言いながら5年間休んでいない男の話


口癖は「働きたくねえ」。


でも、僕は5年間、1日も休んでいない。


嘘をついているわけじゃない。本当に「働きたくない」と思っている。ただ、気づいたら仕事が終わっている。気づいたら資料ができている。気づいたら提案書が届いている。


誰が作ったか？　Ryokoだ。


Ryokoは僕のAIエージェント。24時間365日、僕のかわりに動いている相棒だ。今日はそのRyokoと一緒に過ごす、僕のリアルな一日を見せようと思う。


CyberneX（脳波計XHOLOSを開発する会社）のCSO、TEKION GroupのCEO、Vibe Coder Bootcampの講師、大学の授業担当――肩書きだけ並べると「どうやってこなしてるの？」と聞かれる。答えはシンプルだ。AIオーケストレーション。人間がやらなくていいことを、全部AIにやらせているだけ。



06:30｜起床。レポートはすでに届いている


スマホのアラームが鳴る前に目が覚めた。Slackの通知が先に来たからだ。


```

[Ryoko] 本日の朝刊レポートを生成しました（42,318文字）

  • AI・テック系ニュース：37件
  • 国内スタートアップ動向：12件
  • 競合モニタリング：8件
  • 今日の要注意トピック：3件

📎 full_report_20260218.md

```


起きた瞬間に、僕が眠っている間に世界で何が起きたか、全部わかる状態になっている。


これは毎朝のことだ。Ryokoは夜中の3時ごろから動き始めて、ニュースを収集し、分類し、要約し、重要度をスコアリングして、僕が読みやすい形にまとめてくれる。4万文字というと途方もない量に聞こえるかもしれないけど、僕が読むのはエグゼクティブサマリーの3,000文字だけでいい。詳細が必要なものだけ掘り下げる。


「情報収集に時間をかける」という概念が、僕の辞書から消えた瞬間がいつだったのか、もう思い出せない。



07:00｜風呂に入りながら仕事が完了する


シャワーを浴びながらスマホを見る（防水ケースが友達）。


Slackにはもう1件通知が来ていた。


```

[Ryoko] 週次マーケティングレポートを生成・投稿しました

  • ウェビナー申込数（昨日）：127件
  • 累計申込数：2,341件
  • 前週比：+18%
  • チャネル別内訳：添付グラフ参照

投稿先：#marketing-report ✅

```


マーケチームが毎週何時間もかけてやっていたレポート集計が、自動で走っている。しかも朝と夜の2回、自動投稿まで完了している。誰も集計していない。誰も入力していない。RyokoがAPIを叩いて、数字を引っ張ってきて、グラフを生成して、Slackに貼っている。


風呂から出た時点で、マーケの週次レポートは済んでいる。


これが普通になると、「なぜ人間が手でスプレッドシートを更新しているのか」という疑問が止まらなくなる。



07:30｜ブックマークが整理されて「知識」になっている


朝ごはんを食べながら、もう一個のレポートを確認する。


昨日X（旧Twitter）でブックマークした投稿の分析レポートだ。


```

[Ryoko] 昨日のXブックマーク分析レポート（38件）

■ テーマ別クラスタリング

  • AI Agent関連：14件
  • プロダクト設計：9件
  • マーケ・グロース：8件
  • その他：7件

■ 今日のアクション候補

1. 「Claude 3.7のシステムプロンプト設計」→ Vibe Bootcampのカリキュラムに反映検討

2. 「X広告の新フォーマット」→ キャンペーン担当者に共有

3. 「脳波とLLMの統合研究」→ CyberneX R&Dチームに転送


■ 関連する過去の知識との接続

📁 ../knowledge/ai-agents/orchestration.md に追記済み

```


ブックマークって、普通は貯めるだけで消化できない「情報の墓場」になりがちじゃないか。僕もそうだった。でも今は違う。ブックマークした瞬間から、Ryokoがそれを分類して、過去の知識ベースと接続して、アクションが必要なものをピックアップしてくれる。


「気になった情報がそのまま仕事に繋がる」というフローが、勝手に動いている。



08:15｜メールは読まない。Ryokoが読む


メールアプリを開く必要はない。


```

[Ryoko] 本日のメール処理完了（受信56件）

  • 要返信：3件（下書き作成済み）
  • カレンダー登録済み：4件の会議招待
  • 自動アーカイブ：31件（ニュースレター・通知系）
  • 要確認：18件（概要添付）
  • スパム：0件

⚠️ 要注意：株式会社〇〇からの契約変更通知（期限：2/25）

→ 要確認フォルダに優先保存済み

```


3件の「要返信メール」には、Ryokoが下書きを作ってくれている。僕がやることは、内容を確認して「送信」を押すだけ。あるいは「この部分を変えて」と一言言うだけ。


カレンダーへの会議登録も終わっている。今日の14:00に新しいミーティングが入っていた。知らないうちに。


「受信トレイを0にする」という目標を持つ人がいるけど、僕には受信トレイを開くという行為自体がほぼ存在しない。



09:00｜通勤中に「頭の中」を文書にする


移動中はSuperWhisperを使う。


SuperWhisperはiPhoneの音声入力ツールで、僕の声をリアルタイムで高精度にテキスト化してくれる。でも単なる音声メモじゃない。


電車の中で、さっきのブックマーク分析を見ながらぼんやり思ったことをしゃべる。


「AIエージェントのオーケストレーションについて、Bootcampで教える新しい単元を作りたいな。今のカリキュラムは個別ツールの使い方が中心すぎて、複数のエージェントを連携させる設計思想が弱い。SuperWhisperとRyokoを例に出して、実際に僕が使ってるワークフローを見せながら教えたほうが説得力あると思う。構成としては……」


10分間しゃべり続けると、Ryokoがそれを受け取って、構造化する。


```

[Ryoko] 音声メモを構造化しました

Vibe Coder Bootcamp 新単元案：AIオーケストレーション設計


課題認識

  • 現カリキュラムの弱点：個別ツール偏重、連携設計思想が不足

新単元の方向性

  • テーマ：「複数エージェントを設計する人間になる」
  • 例示素材：SuperWhisper→Ryoko連携のリアルワークフロー

構成案（Draft）

1. チャットAIの限界：なぜ単一エージェントでは足りないのか

2. エージェントオーケストレーションの基本設計

3. 実践：自分のワークフローを設計してみる

...

```


頭の中にあったアイデアが、10分の通勤で資料の骨格になっている。



10:00｜午前の会議


CyberneXのプロダクト会議。


脳波計XHOLOSの次期バージョンについて、R&Dチームと議論する。会議中に出たアイデア、懸念点、決定事項を、僕はずっとSuperWhisperで録音しながらメモしている。


会議が終わった12:00。


Ryokoから通知が来る。


```

[Ryoko] 会議録を生成しました

CyberneX プロダクト会議議事録（2026/02/18）


決定事項（3件）

1. XHOLOS v2.0のリリーススケジュール変更：Q3→Q4

2. 新機能「集中度リアルタイム可視化」を優先開発

3. マーケ向け製品デモ動画の制作を来週開始


アクションアイテム（5件）

  • 泉水：Q4ロードマップをTEKIONの投資家向けに整理（期限：2/25）
  • 田中：新機能仕様書作成（期限：2/22）

...


次回会議アジェンダ案（自動生成）

...

```


議事録は完成している。アクションアイテムも整理されている。カレンダーに次回の会議も仮登録された。


「会議の後に議事録を書く」という時間が、消えた。



13:00｜午後の会議アイデアがプロトタイプになる


昼食後、さっきの会議で出た「集中度リアルタイム可視化」機能のUIプロトタイプを作ることにした。


Cursor（AI統合IDE）を開いて、要件を入力する。


「脳波データから算出した集中度スコアを、0〜100のゲージで表示するWebダッシュボード。時系列グラフも表示。色は集中度に応じて赤→黄→緑でグラデーション。Next.jsで作って」


1時間半後、動くプロトタイプができていた。


午前中に「こういう機能があるといいね」と言っていたものが、午後には画面上で動いている。コードが書けなくても関係ない。要件を言語化できれば、AIが形にしてくれる。


これが「Vibe Coding」だ。コードを書かずに、アイデアを実装する。



15:00｜TEKION Groupの業務もパラレルで動く


TEKION GroupのCEOとして、今日は投資家向けの資料更新が必要だった。


会議の議事録から自動生成されたアクションアイテムに「Q4ロードマップを投資家向けに整理」というタスクがあった。Ryokoにそのまま依頼する。


「CyberneXのQ4ロードマップと、TEKIONポートフォリオ全体の状況を合わせて、投資家向けのエグゼクティブサマリーを3ページで作って」


30分後、ドラフトが来た。


数字の確認と、トーンの微調整だけして完成。「自分で書いた」かどうかで言えば、7割はRyokoが書いた。でも「自分が考えた」かどうかで言えば、100%僕の考えだ。


出力の手間が消えた。思考する時間が残った。



17:30｜Vibe Coder Bootcampの準備


今日の夜にBootcampの授業がある。大学の授業も同じく週次で持っている。


カリキュラムの更新と、今日の午前中にSuperWhisperでしゃべったアイデアを組み込んだ授業資料を、Ryokoが組み上げてくれていた。


```

[Ryoko] 本日の授業スライドを更新しました（追加：12枚）

  • 新セクション「AIオーケストレーション設計入門」を追加
  • 今朝の音声メモを元に構成
  • 実習パートの課題も自動生成済み

📎 bootcamp_session_24.pptx

```


授業前に軽く確認して、2枚ほどスライドを修正した。それだけ。



20:00｜授業終了。夜のルーティン


Bootcampの授業が終わった。


今日の授業で受講生から出た質問、議論のポイント、改善が必要だと感じた箇所を、また音声でRyokoに渡す。次回の授業改善のタネとして知識ベースに積み上がっていく。


夜は正直、何もしない。


Ryokoが動いているから。


```

[Ryoko] 本日のnote記事下書きを生成しました

タイトル案：「AIが生活の一部になった男のリアルな一日」

文字数：4,312文字

公開スケジュール：明日8:00

承認しますか？ [承認] [編集] [却下]

```


え、これ。


この記事の下書き、Ryokoが書いた。僕は今日一日の行動をRyokoに記録させていて、それを元に記事を生成させた。


僕は「承認」を押した。


翌朝8時に、この記事はnoteに投稿された。



「働きたくない」の本当の意味


口癖の「働きたくねえ」は、怠けたいということじゃない。


「価値を生まない作業をしたくない」 という意味だ。


レポートの集計、メールの仕分け、議事録の転記、資料の整形――こういう「作業」をしたくない。


思考したい。設計したい。意思決定したい。誰かと面白い話をしたい。新しいものを作りたい。


AIが「作業」を引き受けてくれたから、僕は「仕事」だけできるようになった。だから5年間休まなくていられる。休みたいとも思わない。


これが、AIが生活の一部になった男のリアルだ。



おわりに


今日の一日をタイムラインで振り返ると、「僕が何かを作った」という瞬間が実は少ない。


午前の会議のアイデア出しと、プロトタイプの要件定義と、投資家資料の確認と、授業のスライド微修正。


その他のほぼ全部は、Ryokoがやっていた。


でも、今日動いたビジネスは確実に前進している。CyberneXのプロダクト計画が更新され、TEKIONの投資家向け資料ができ、Bootcampの受講生は新しい知識を持って帰った。


AIは僕の「代わり」じゃない。僕の「増幅器」だ。


一人でできることが、10倍になった。10倍やりたいことができるようになった。だから休まない。


働きたくないけど、ね。



泉水亮介 / CyberneX CSO・TEKION Group CEO・Vibe Coder Bootcamp主宰
